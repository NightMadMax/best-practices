# Best Practices

## Цель

- Сохранять лучшие практики, уроки и типовые ошибки из завершённых проектов,
  чтобы они переиспользовались в новых проектах и разными командами.

## Scope

- Практики по 1С, вебу и общие практики в `practices/`.
- Выжимки уроков из проектов перед их удалением.
- Командное использование через общий GitHub-репозиторий и pull requests.

## Non-goals

- Хранение кода проектов, документации конкретных проектов, секретов.
- Дублирование данных, у которых есть первичный машиночитаемый источник.

## Пользователи

- Max и команды, начинающие новые проекты на основе шаблона.
- AI-агенты, читающие практики при старте похожего проекта.

## Ограничения

- Только Markdown внутри общего Obsidian vault; правки в общий репозиторий —
  через PR.

## Критерии успеха

- При старте похожего проекта прошлые уроки применяются без повторного
  наступания на те же грабли.
