# ADR-0002. Вердикты инструментов — по кольцам tech-radar

- **Статус:** accepted (2026-07-05)

## Context

Каталог инструментов и MCP-серверов ([[practices/tools/README]]) должен не просто
перечислять, а подсказывать, что брать. Простой список не отражает степень
доверия и приводит к спорам «использовать или нет».

## Decision

Каждому инструменту/серверу присваивается вердикт по модели tech-radar
(ThoughtWorks): **Adopt / Trial / Assess / Hold**.

- **Adopt** — проверено на нескольких проектах, брать по умолчанию.
- **Trial** — применять на реальных задачах, с оглядкой.
- **Assess** — исследовать, пока не в проде.
- **Hold** — не начинать новое; тяжёлые случаи раскрывать в
  [[practices/anti-patterns/README]].

Крупные выборы «X вместо Y» оформляются отдельным ADR, на который ссылается
строка каталога.

## Consequences

- Плюс: сразу видно, что внедрять, что пробовать, чего избегать.
- Плюс: вердикт эволюционирует с опытом и привязан к проекту-источнику
  (см. [[docs/architecture/decisions/ADR-0001-provenance-required]]).
- Минус: нужно поддерживать актуальность колец при новом опыте.
