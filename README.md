# Best Practices

## Назначение

База знаний с лучшими практиками, извлечёнными из завершённых проектов.
Проекты приходят и уходят — уроки остаются здесь.

## Структура

| Раздел | Назначение |
|---|---|
| `practices/1c/` | Практики разработки для 1С (модули, типовые ошибки, чек-листы) |
| `practices/web/` | Практики веб-разработки |
| `practices/common/` | Общие практики, не привязанные к стеку |

## Как пользоваться

1. **Начало нового проекта** — открой релевантный раздел `practices/` или
   попроси агента: «прочитай практики по 1С из Best Practices».
2. **Завершение проекта** — перед удалением попроси агента:
   «сделай выжимку уроков из этого проекта и добавь в Best Practices»
   (см. skill `harvest-project-lessons` в проекте «Правила для нового проекта»).
3. **Командная работа** — правки через pull request в GitHub-репозиторий,
   чтобы база проходила ревью и не превращалась в свалку.

## Формат записи практики

Каждая практика — отдельный раздел в тематическом файле: контекст (когда
применимо), проблема, решение, источник (из какого проекта пришло).

## Правила

- Эта папка является git-root проекта и находится внутри родительского
  Obsidian vault.
- Markdown редактируется напрямую, без API, helper и второй копии.
- Машиночитаемые спецификации остаются первичными источниками истины.
