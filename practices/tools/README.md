# Практики: инструменты и MCP-серверы

Кросс-проектный каталог инструментов (CLI, библиотеки, сервисы) и MCP-серверов:
что использовали, **для чего** и стоит ли брать снова. В отличие от `TOOLS.md`
внутри проекта, здесь копится опыт по многим проектам.

Оформление вердикта — по модели tech-radar (ThoughtWorks).

## Кольца (вердикт)

- **Adopt** — проверено на нескольких проектах, брать по умолчанию.
- **Trial** — стоит применять на реальных задачах, с оглядкой.
- **Assess** — присмотреться/исследовать, пока не в проде.
- **Hold** — не начинать новое на этом; причину указать в заметках
  (тяжёлые случаи выносить в [[practices/anti-patterns/README]]).

Происхождение обязательно для каждой строки: **Источник** — проект, откуда
пришло, **От кого** — кто предложил/подтвердил (пользователь, автор или агент).
См. [[docs/architecture/decisions/ADR-0001-provenance-required]].

## MCP-серверы

| Сервер | Для чего | Вердикт | Настройка (без секретов) | Источник | От кого | Заметки |
|---|---|---|---|---|---|---|
| _пример_ context7 | Подтягивать актуальные доки библиотек в контекст агента | Trial | npx, ключ через env-переменную | `jira-analytics` | Max | Экономит на «залипании» на устаревших API |

## Инструменты (CLI, библиотеки, сервисы)

| Инструмент | Для чего | Вердикт | Настройка (без секретов) | Источник | От кого | Заметки |
|---|---|---|---|---|---|---|
| _пример_ ripgrep (`rg`) | Быстрый поиск по коду вместо `grep -r` | Adopt | входит в окружение | `common` | агент | Дефолт для поиска в репозитории |

## Правила каталога

- Одна строка — один инструмент/сервер; не дублировать, а обновлять вердикт.
- **Никаких секретов**: токены, ключи, приватные URL и хосты не записывать —
  только способ конфигурации (env-переменная, файл, флаг).
- Вердикт менять при новом опыте и указывать проект-источник изменения.
- Для MCP держать ссылку на официальный источник сервера, а не копию описания.
