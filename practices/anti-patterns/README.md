# Практики: анти-паттерны

Чего избегать и почему — зеркало [[README|практик]]. Подходы, инструменты и
решения, которые на практике себя не оправдали. Тяжёлые «Hold» из
[[practices/tools/README]] раскрываются здесь подробно.

## Формат записи

### <Название анти-паттерна>

- **Контекст:** где встречается, когда соблазнительно так сделать.
- **Почему плохо:** что ломается, какой ценой (со ссылкой на симптом).
- **Что делать вместо:** проверенная альтернатива.
- **Источник:** проект, из которого пришёл урок.
- **От кого:** пользователь (имя/роль), автор коммита или агент.
- **Дата:** когда добавлено или подтверждено.
